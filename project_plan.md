# HireMatch AI PRD

## 1. Product Summary
**Elevator Pitch**: A specialized AI tool for evaluating resume fit against a custom job description. Instead of generic ATS resume scoring, this tool provides a detailed match score, skill gap analysis, and generates technical/behavioural interview questions specifically tailored to the candidate’s gaps.

**Key Decisions**:
*   No user accounts or auth in v1 — completely stateless, no login required
*   No bulk processing or ATS integration in v1
*   No mobile app — web only

## 2. Target Users

| Persona | Who They Are | What They Need |
| :--- | :--- | :--- |
| **Job Seeker** | Developer or professional applying to roles | Know how well their resume matches a JD before applying; understand what to add |
| **Recruiter** | HR or hiring manager screening candidates | Quick signal on candidate fit; ready-made interview questions |
| **Interviewer** | Engineer doing a technical screen | Relevant questions auto-generated from the resume and JD |

## 3. Feature Specification

### 3.1 Input Panel — Job Description + Resume Upload
*   Large textarea for pasting the job description (no character limit)
*   PDF upload button for the candidate resume — drag and drop supported
*   Optional: plain text fallback if user wants to paste resume instead of uploading PDF
*   'Analyze' button — disabled until both inputs are provided
*   Character/word count shown below JD textarea so user knows it was read

### 3.2 Match Score Card — Hero Feature
This is the first thing the user sees after clicking Analyze. It must feel premium and immediate.
*   Large circular score meter: 0–100, color coded (red < 40, yellow 40–70, green > 70)
*   Score breakdown sub-scores: Skills Match, Experience Relevance, Education Fit, Keywords Coverage
*   One-line verdict below the score: e.g. 'Strong Match — ready to apply' or 'Needs work — 4 gaps found'
*   Animated score counter on load (counts up from 0 to final score for visual impact)

### 3.3 Gap Analysis Panel
*   Two-column layout: 'Skills You Have' (green tags) vs 'Skills Missing' (red tags)
*   Each missing skill has a tooltip explaining why it matters for this specific role
*   'What to add to your resume' section: 3-5 bullet point suggestions generated by AI
*   Priority ranking: Critical gaps vs Nice-to-have gaps

### 3.4 AI-Generated Interview Questions — Key Differentiator
This is the feature that makes recruiters want to use the tool themselves.
*   10 questions generated per analysis, split into categories:
    *   **Technical (5 questions)** — based on skills in the JD the candidate claims to have
    *   **Behavioural (3 questions)** — based on role responsibilities
    *   **Gap Probing (2 questions)** — specifically targeting identified skill gaps
*   Each question has a collapsible 'What a good answer looks like' hint
*   'Copy all questions' button for recruiters to paste into their interview doc

### 3.5 Resume Improvement Suggestions
*   Side-by-side view: original resume text on left, suggested rewrites on right
*   AI rewrites weak bullet points to be more JD-aligned (e.g. adds relevant keywords)
*   'Apply suggestion' button copies the improved bullet to clipboard
*   Maximum 5 suggestions to keep it focused and not overwhelming

### 3.6 Export & Share
*   'Download Report' button — generates a clean PDF summary of the analysis
*   Shareable link: analysis result stored temporarily (24 hours) with a unique URL
*   'Copy link' button so a job seeker can share their analysis with a mentor or recruiter

## 4. Technical Architecture

### 4.1 Tech Stack
| Layer | Technology | Why |
| :--- | :--- | :--- |
| **Frontend** | React + TypeScript + Tailwind CSS | Consistent with CollabSpace stack; TS already on resume |
| **PDF Parsing** | pdf-parse (Node.js) | Extracts raw text from uploaded resume PDF server-side |
| **AI Engine** | Gemini API (gemini-2.0-flash) | Cost-effective, fast, and supports large context windows — ideal for resume + JD analysis |
| **Backend** | Node.js + Express + TypeScript | Handles file upload, PDF parsing, and Gemini API calls |
| **File Upload** | Multer (Express middleware) | Handles multipart form data for PDF upload |
| **Temp Storage** | MongoDB (24hr TTL index) | Stores analysis results for shareable links; auto-deletes |
| **PDF Export** | Puppeteer or jsPDF | Generates downloadable report from analysis result |
| **Deployment** | Vercel (FE) + Render (BE) | Same pattern as CollabSpace — already know how to do this |

### 4.2 Core API Flow
The entire value of this app happens in one API call. Everything else is UI.
1.  **User submits form**: Frontend sends PDF file + JD text to POST /api/analyze
2.  **PDF parsing**: Multer receives file, pdf-parse extracts raw resume text
3.  **Prompt construction**: Backend builds a structured prompt with JD + resume text
4.  **Gemini API call**: Send prompt, instruct Gemini to return JSON with score, gaps, questions, suggestions
5.  **Parse response**: Parse Gemini JSON response, validate structure, handle errors
6.  **Store result**: Save to MongoDB with 24hr TTL, generate unique shareId
7.  **Return to frontend**: Send full analysis JSON to frontend for rendering

### 4.3 Gemini Prompt Design
The quality of this entire product depends on one well-engineered prompt. This is the most important technical decision in the project.
*   Instruct Gemini to return ONLY valid JSON — no markdown, no preamble
*   Define the exact JSON schema in the prompt with field names and types
*   Include few-shot examples of good scoring logic so Gemini calibrates correctly
*   Separate the prompt into clearly labelled sections: ROLE, JD, RESUME, INSTRUCTIONS
*   Ask for scores as integers 0-100, gaps as arrays of strings, questions as objects with { question, category, hint }
*   Add a 'confidence' field so low-quality resumes get flagged rather than confidently mis-scored

*Example JSON schema to request from Gemini:*
\`\`\`json
{ "overallScore": 78, "breakdown": { "skillsMatch": 85, "experienceRelevance": 72, "educationFit": 80, "keywordsCoverage": 75 }, "verdict": "Strong Match", "skillsFound": [...], "skillsMissing": [...], "suggestions": [...], "interviewQuestions": [...] }
\`\`\`

### 4.4 Data Models
**Analysis Result (MongoDB — 24hr TTL)**
*   _id, shareId (uuid), jdText, resumeText, overallScore, breakdown, skillsFound[], skillsMissing[], suggestions[], interviewQuestions[], createdAt (TTL index)

## 6. Success Criteria
*   Upload any real PDF resume + paste any real JD → get a result in under 10 seconds
*   Score feels accurate — a strong match should score 70+, a weak match should score below 50
*   Interview questions are specific to the role and resume — not generic
*   Zero visible errors or broken UI when demoing to a recruiter
*   Shareable link works when opened in incognito — analysis loads without re-uploading
*   README includes a demo GIF showing the full flow in under 30 seconds
